Start Simulator

Use Case 1 Start Simulator
Goal in Context User wants to start simulator to run simulation.
Scope & Level Simulator, Summary
Preconditions User has simulator properly installed.
Success End Condition Simulator Unity server is running with UI.
Failed End Condition Simulator
Primary, Secondary Actors Normal user of this simulator
Trigger User double clicks simulator execution file.
DESCRIPTION Step Action
1 User navigates to simulator installed folder.
2 User clicks the execution file inside this folder
3 Simulator runs initial checks for making sure all requirements are met
4 Simulator starts with UI with no error messages showing

Select Robot

Use Case 2 Select Robot
Goal in Context User wants to select a robot to run simulation.
Scope & Level Simulator, Sub function
Preconditions Simulator is running, world/maze environment has been created.
Success End Condition Simulator gets the robot model selected by the user and places the robot at user-intended position.
Failed End Condition Simulator doesn’t get the robot model selected by the user, simulator doesn’t place the robot as intended by the user.
Primary, Secondary Actors Normal User of this simulator
Trigger User clicks the “add robot” button and select one robot using file picker.
DESCRIPTION Step Action
1 User clicks the “add robot” button.
2 User select one of the robot by clicking one in the default robot folder.
3 The selected robot will stick to the cursor, user will put the robot in the desired position of the environment
4 User can also specify a coordinate of the desired position by inputting an x and y value.
5 Simulator checks the provided coordinates or user cursor position and initialize the robot to the nearest possible position (incase the coordinates conflict with the walls )
6 User checks the position of the robot
7 User can relocate the robot to another spot after placed.
EXTENSIONS Step Branching Action
5a X or y value is too large (exceeds the dimensions of floor) or x or y is not a number.
5a1 Simulator show error messages
5b User mouse cursor with robot is beyond the limits of environment when user clicks mouse button to place the robot.
5b1 The robot becomes red, indicating the wrong place to put the robot.
6a User want to delete the created robot by double clicking the robot and click “delete robot” button.
6a1 Use Case 3
6b User inspect the current robot position by double clicking the robot.
6b1 Use Case 4
6c User add client to the current robot by double clicking the robot and click “add client” button.
6c1 Use Case 5
7a User relocate the robot by double clicking the robot and input a new X and Y value.
7a1 Use Case 6

DESCRIPTION Step Action
1 User opens the inspector panel by double clicking the robot.
2 User put in new coordinate values or phi value.
3 User clicks “relocate” button on the inspector panel.
5 Simulator checks the input.

5a X or y value is too large (exceeds the dimensions of floor) or x or y or phi is not a number.

Start Simulation

Use Case 7 Start Simulation
Goal in Context User wants to start simulation with customized or demo world/maze file, and customized or demo script file.
Scope & Level Simulator, Primary Task
Preconditions Simulator is running, simulator has got the user-intended world/maze file, robot has been selected by user, client has been attached to the robot.
Success End Condition The simulator runs simulation according to script, client window is showing based on user control
Failed End Condition Simulator doesn’t run simulation according to script. Simulator doesn’t show client window on user control. Client window doesn’t work properly
Primary, Secondary Actors Normal User of this simulator
Trigger User click “start ” button on simulator UI
DESCRIPTION Step Action
1 User click “start simulation” button on UI to start the simulation
2 Simulator runs simulation according to the script
EXTENSIONS Step Branching Action
1a User double click on the robot and inspector window is showing
1a1 Simulator pops up a client window showing LCD screen and 4 buttons
1a2 User can interact with robot by clicking on the 4 buttons if programed in the script.
1a3 Robot behaves responding to client button clicking.

Add Errors

Use Case 8 Add Errors
Goal in Context User wants to add errors to simulation to make the simulation more realistic
Scope & Level Simulator, Sub function
Preconditions Simulator is running, simulator has got the user-intended world/maze file, robot has been selected by user, script file and client has been attached to the robot.
Success End Condition Simulator runs the simulation with error as specified by the user.
Failed End Condition Simulator doesn’t run the simulation with error as specified by the user.
Primary, Secondary Actors Normal User of this simulator
Trigger User clicks the “add error” button on UI
DESCRIPTION Step Action
1 User clicks the “add error” button on UI
2 Simulator shows a list of errors for user to select
3 User select one or more of the errors in the list to add
4 User input the percentage of error in the input box on the side of each error.
5 Simulator applies the errors and percentage for each error to the respective simulation algorithms
6 Simulator show “add error successfully” message

Execute Simulation Script

Use Case 15 Execute Simulation Script
Goal in Context User wants to run simulation in batch mode by typing command in terminal.
Scope & Level Simulator, Summary
Preconditions Simulator should be running, user has access to terminal, user has prepared simulation file containing all required information like path to word/maze file, and robot file, etc.
Success End Condition Simulator Unity server runs the simulation file, and create world/maze environment, select robots ,assign script and client accordingly, and runs the script correctly.
Failed End Condition Simulator can’t handle the simulation file properly. Simulator doesn’t show any error message when the file contains problems. Simulator doesn’t correctly run the script file.
Primary, Secondary Actors Advanced user of this simulator
Trigger User type in command line
DESCRIPTION Step Action
1 User types batch command in terminal specifying the prepared simulation file.
2 The simulation file get checked by simulator and fed back with any error if encountered.
3 The simulator executes the script file specified in the simulation file correctly.
4 Simulator stops simulation after script is finished.
